# เว็บติดตามใบกำกับภาษีที่ยังไม่ได้รับ — BENZ

ระบบสำหรับทีมภายใน ใช้บันทึก/ติดตามรายการซื้อที่ยังไม่ได้รับใบกำกับภาษีจากผู้ขาย พร้อมวันที่คาดว่าจะได้รับ
สถานะ และ aging (ค้างนานแค่ไหน) — ทุกคนที่ login แล้วมีสิทธิ์เข้าถึงข้อมูลเท่ากัน (equal access)

Stack: Next.js 16 (App Router, client-heavy) + Supabase (database + auth) + Tailwind CSS
ออกแบบให้ deploy บน Vercel

> |---|---|

สำหรับผู้ใช้ที่ไม่ได้ login (anon) เลย

## 2. รันในเครื่องตัวเอง


## 3. Deploy ขึ้น Vercel

วิธีที่แนะนำ (ไม่ต้องแชร์ API token ให้ใคร):

1. สร้าง repository บน GitHub แล้ว push โค้ดโปรเจกต์นี้ขึ้นไป:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin <URL ของ repo คุณ>
   git push -u origin main
   ```
2. เข้า [vercel.com/new](https://vercel.com/new) แล้วกด **Import** repository ที่เพิ่ง push ขึ้นไป
   — Vercel จะตรวจพบว่าเป็น Next.js โปรเจกต์เองอัตโนมัติ ไม่ต้องตั้งค่า build command เพิ่ม
3. ก่อนกด Deploy ให้เพิ่ม **Environment Variables** ในหน้าตั้งค่าของ Vercel:
   | Name | Value |
   |---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | Project URL จาก Supabase (ขั้นตอนที่ 1) |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | anon/public key จาก Supabase (ขั้นตอนที่ 1) |
4. กด **Deploy** — ครั้งต่อไปที่ push โค้ดขึ้น branch `main` จะ deploy ให้อัตโนมัติทุกครั้ง

## 6. นำเข้ารายการจาก Excel

หน้า dashboard มีปุ่ม **"นำเข้าจาก Excel"** ข้าง ๆ ปุ่ม "+ เพิ่มรายการ" สำหรับบันทึกหลายรายการพร้อมกัน
โดยไม่ต้องกรอกทีละรายการผ่านฟอร์ม

ขั้นตอนใช้งาน:

1. กด **"ดาวน์โหลดเทมเพลต Excel"** ในแผงนำเข้า — ได้ไฟล์ `.xlsx` ที่มีหัวคอลัมน์ถูกต้องพร้อมแถวตัวอย่าง 1 แถว
2. กรอกข้อมูลต่อจากแถวตัวอย่าง (หรือลบแถวตัวอย่างทิ้งแล้วกรอกของจริงแทน) แล้วบันทึกไฟล์
3. กด **"เลือกไฟล์ Excel..."** เลือกไฟล์ที่กรอกแล้ว — ระบบจะตรวจสอบทุกแถวและแสดงตารางตัวอย่างทันที
   พร้อมสถานะรายแถว (พร้อมนำเข้า / มีปัญหา พร้อมเหตุผล)
4. ตรวจสอบตัวอย่างแล้วกด **"นำเข้า N รายการ"** — จะบันทึกเฉพาะแถวที่ผ่านการตรวจสอบเท่านั้น
   (แถวที่มีปัญหาจะถูกข้ามไป ไม่ต้องแก้ทุกแถวให้ถูกก่อนถึงจะนำเข้าแถวอื่นได้)

หัวคอลัมน์ในเทมเพลต:

| คอลัมน์ | จำเป็นไหม | หมายเหตุ |
|---|---|---|
| ผู้ขาย | จำเป็น | |
| วันที่ทำรายการ | จำเป็น | รองรับเซลล์วันที่จริงของ Excel, `YYYY-MM-DD`, หรือ `DD/MM/YYYY` |
| รายละเอียด | ไม่จำเป็น | |
| ยอดก่อน VAT | จำเป็น | ต้องเป็นตัวเลขมากกว่า 0 (ใส่ comma คั่นหลักพันได้ เช่น `1,000`) |
| VAT | ไม่จำเป็น | **เว้นว่างไว้ระบบจะคำนวณให้อัตโนมัติ 7%** จากยอดก่อน VAT (เหมือนฟอร์มเพิ่มรายการทีละตัว) |
| เลขที่อ้างอิง | ไม่จำเป็น | |
| วันที่คาดว่าจะได้รับใบกำกับภาษี | ไม่จำเป็น | ถ้ากรอกต้องไม่ก่อนวันที่ทำรายการ |
| หมายเหตุ | ไม่จำเป็น | |

แถวที่ว่างทั้งแถว (เช่นแถวว่างท้ายไฟล์) จะถูกข้ามไปอัตโนมัติ ไม่ถือเป็นปัญหา

> **หมายเหตุด้านเทคนิค:** ใช้ไลบรารี `xlsx` (SheetJS) อ่าน/เขียนไฟล์ Excel ฝั่ง client ทั้งหมด — ไม่มีการ
> ส่งไฟล์ไปประมวลผลที่ server ใดๆ `npm audit` แจ้งช่องโหว่ระดับ high ในไลบรารีนี้ (prototype pollution,
> ยังไม่มี patch) แต่ประเมินแล้วว่าความเสี่ยงต่ำสำหรับการใช้งานนี้ เพราะเป็นเครื่องมือใช้ภายใน
> ประมวลผลเฉพาะไฟล์ที่ผู้ใช้ในทีมอัปโหลดเอง และผลกระทบจำกัดอยู่แค่ browser tab ของผู้ใช้เอง
> (ไม่กระทบฐานข้อมูลหรือผู้ใช้คนอื่น) — ทางเลือกอื่น (`exceljs`) มี dependency tree ที่หนักกว่าและ
> deprecated มากกว่าโดยไม่ได้ลดความเสี่ยงจริงลงในบริบทนี้

## 7. เมนู Sidebar

หน้าเว็บมีแถบเมนูด้านซ้าย (Sidebar) แบบระบบ ERP ครอบคลุมทั้งเว็บ — เนื้อหาเดิม (หน้าติดตาม
ใบกำกับภาษี, สถิติ, ตาราง, สรุป VAT รายเดือน ฯลฯ) ไม่มีการเปลี่ยนแปลง Logic/ฟังก์ชันใด ๆ เลย
แค่ถูกย้ายไปแสดงในเมนู **"บันทึกค่าใช้จ่าย"** (อยู่ในหมวด "บันทึกการจ่ายเงิน" — เป็นหน้าแรกของระบบ
ตอนนี้ active เป็นค่าเริ่มต้นเสมอ และหมวด "บันทึกการจ่ายเงิน" ที่เมนูนี้อยู่จะถูกขยายให้อัตโนมัติ
ตั้งแต่ล็อกอินครั้งแรกด้วย)

โครงสร้างเมนู:

- **บันทึกการจ่ายเงิน** — บันทึกค่าใช้จ่าย (หน้าเดิมทั้งหมด มีฟีเจอร์จริง เป็นค่าเริ่มต้น), รายงานจ่ายเงิน
- **กระทบยอด** — Bank Reconcile, VAT Reconcile, ภาษีซื้อไม่ถึงกำหนด, ตรวจสอบข้อมูล

เมนูย่อยนอกเหนือจาก "บันทึกค่าใช้จ่าย" ยังไม่มีฟีเจอร์จริง คลิกได้ตามปกติแต่จะขึ้นหน้า **"เร็วๆ นี้"**
แทน (ไม่ใช่ปุ่มที่กดแล้วไม่มีอะไรเกิดขึ้น) — เพิ่มฟีเจอร์จริงทีหลังได้โดยไม่ต้องแก้ Sidebar เลย
แค่สร้างหน้า/component ใหม่แล้วเปลี่ยน `implemented: false` เป็น `true` ที่ `lib/navigation.ts`

> **หมายเหตุเรื่อง Route:** เว็บนี้ไม่มี URL แยกต่อเมนู — ทุกเมนูอยู่ใน URL เดียวคือ `/dashboard`
> แล้วสลับเนื้อหาด้วย state ฝั่ง client (จำเมนูที่เลือกไว้ใน localStorage) เดิมเมนูแรกสุดชื่อ
> "ใบกำกับภาษี" (id: `dashboard`) — ตอนนี้เมนูนั้นถูกลบออกจาก Sidebar แล้ว และหน้าเนื้อหาเดิมของมัน
> (สถิติ/ตาราง/filter/นำเข้า Excel/สรุป VAT รายเดือน) ถูกย้ายไปผูกกับเมนู "บันทึกค่าใช้จ่าย"
> (id: `record-expense`) แทน โดยไม่ได้สร้างหน้าใหม่หรือย้าย logic ใด ๆ เลย (ดู `lib/navigation.ts`)

พฤติกรรมอื่น ๆ: หมวด "บันทึกการจ่ายเงิน" และ "กระทบยอด" ยุบ/ขยายได้อิสระจากกัน, ระบบจำเมนูที่
เลือกและสถานะยุบ/ขยายไว้ในเบราว์เซอร์ (ข้าม refresh ได้ แต่เป็นการจำต่อเบราว์เซอร์ ไม่ใช่ต่อบัญชี),
และเมื่อจอแคบกว่า 992px (มือถือ/แท็บเล็ต) sidebar จะซ่อนเป็นค่าเริ่มต้น เปิดผ่านปุ่มแฮมเบอร์เกอร์มุม
บนซ้าย ปิดได้ด้วยการแตะพื้นหลังมืด, ปุ่ม X, ปัดนิ้วไปทางซ้ายบน sidebar, หรือเลือกเมนูใด ๆ
